# 4.0.8 夸夸重构

# 优化背景&概述

*   对旧有夸夸功能进行重构：原文档链接：[《3.9.8 夸夸》](https://alidocs.dingtalk.com/i/nodes/o14dA3GK8g5yvg9BHkp7B5wgV9ekBD76?iframeQuery=utm_source%3Dportal%26utm_medium%3Dportal_recent)
    
*   重构概述：
    

1.  现有夸夸遮挡黑点区域，阻挡玩家看刚刚划出的答案；
    

*   优化：需要下移，且尺寸要缩小一些；
    

1.  现有夸夸的图文无分类，对玩家操作的缺乏针对性夸奖，夸奖内容虽多，但每种夸奖的定位较模糊；
    

*   优化：将现有夸夸进行重新分类、重新设定夸夸形式，让玩家将特定操作 与 对应的图&文 产生一一对应的的印象。
    
    1.  极限情况下，划出超级难找的词
        
        *   夸夸条件：在剩余答案>5个单词时（划出之前），划出场上的唯一自由解，且该答案词是逆序词 或 长度<5；
            
        *   夸夸形式：
            
            *   划词前：预告“挑战！”；
                
            *   划词后：夸奖“高端操作！”
                
            *   UI：新形式：UI从右侧滑入屏幕；（预告和夸夸的位置&UI不同）
                
    2.  划错多次后划出一个答案
        
        *   夸夸条件：错了很多次，>5次；
            
        *   形式：现有的横幅的图文形式，用某个特定表情包+特定本地化池子
            
    3.  间隔很长时间后划出一个答案
        
        *   夸夸条件：距离上一次划出答案时间隔了很长，>45s；
            
        *   形式：现有的横幅的图文形式，用某个特定表情包+特定本地化池子
            
    4.  Combo连划答案：
        
        *   夸夸条件：连续划出答案词，中间没有错词操作，只夸答案词，不夸Bw；
            
        *   形式：延续之前的Combo效果；针对答案词的连划进行鼓励词+口播夸奖；
            
    5.  普通情况下，划出难词
        
        *   夸夸条件：在剩余答案>5个单词、且自由解数量>1时（划出之前），划出逆序答案词；
            
        *   夸夸形式：
            
            *   划词后：在划出的单词的松手字母上方，上漂一个大拇指；
                

1.  降低夸夸频率：
    
    *   通过以上方式，提高夸夸门槛，从而降低夸夸频率；
        
    *   仅对答案词进行夸夸，BonusWord不夸夸；
        
2.  其他：替换现有Combo夸夸的incredible口播音效；

# 夸夸开关

## 总开关

*   新夸夸需设置功能模块开关，在FunctionConfig表内新增id=53；
    

| 序号 | 激活关卡序号 | 是否只在主界触发 | 功能是否开启(A用户) | 功能是否开启(B用户) | 功能备注 |
| --- | --- | --- | --- | --- | --- |
| ID | Level | InHomepage | A | B | note |
| int | int | int | int | int | string |
| 53 | 40019 |  | 1 | 1 | 局内夸夸 |

## 子开关

*   新夸夸将夸夸归类，每一类均在夸夸配置表PraiseTypeConfig表内设置独立的开关字段Switch；
    
    *   1=开；
        
    *   0=关；
        

## 用户开关

*   用具设置界面内的夸夸开关仍存在，可控制本次更新后的夸夸是否展示；
    
*   但并非新分类下的所有子分类都受此开关控制：
    
    *   子分类夸夸是否受用户设置控制，需读取PraiseTypeConfig表内isUserControl字段配置：1=受控；0=不受控；
        

## 夸夸展示条件

*   综合下来，夸夸的展示须同时满足以下条件：
    
    *   总开关开启；
        
    *   子开关开启；
        
    *   若受用户开关控制，则用户必须在设置中打开夸夸效果；
        
    *   满足关卡配置条件：
        
        *   LevelConfig表DiffTag>0；
            
    *   满足触发本夸夸的划词条件；
        
        *   客观触发条件；
            
        *   FirstCD；
            
        *   CD
            
    *   满足优先级条件：
        
        *   优先级独立于其他条件；或
            
        *   优先级不独立，但没有优先级更高的夸夸类型满足展示条件；

# 夸夸新配置  PraiseTypeConfig

*   之前的夸夸配置作废：PraiseTierConfig 和 PraiseTextGroupConfig；
    
*   创建新的夸夸配置表：PraiseTypeConfig
    
*   字段解读：
    

1.  FirstCD = 某类型夸夸在单局内首次出现的限制条件，须同时满足：
    
    1.  第一位：间隔划出答案数量
        
        *   开局从0算起，之后划出每一个答案词计+1，须>=配置数字才算满足条件；
            
    
    *   第二位：间隔时长(s)
        
        *   开局从0s算起，间隔的时长须>=配置数字才算满足条件；
            
2.  CD = 两次同类夸夸之间的间隔条件，须同时满足：
    
    1.  第一位：间隔划出答案数量
        
        *   从前一次划词后的夸夸之后开始，从0算起，之后划出每一个答案词计+1，须>=配置数字才算满足条件；
            
    2.  第二位：间隔时长(s)
        
        *   从前一次划词后的夸夸之后开始，从0s算起，间隔的时长须>=配置数字才算满足条件；
            
3.  Priority = 展示优先级
    
    *   配置正数的类型：
        
        *   同时只能展示其中一种；
            
        *   数字越小、优先级越高；
            
    *   配置=-1
        
        *   独立于其他的类型，不比较优先级，满足条件就展示；
